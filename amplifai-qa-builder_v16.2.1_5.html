v16.2.1 current


Bug Fixes


    

  • Import no longer clears card content — refreshPreview was firing during session restore before card fields were populated, causing a blank re-render that wiped visible content; a restore guard flag now prevents preview refresh until all card state is fully applied
    • 

  • Preview updates on Q ID change — changing a card's Q ID now refreshes its own preview; dependency card previews also update when a source card's Q ID changes
    • 

  • Test Runner column tooltip — full task text now shows on hover over truncated column headers (title moved to the <th> element for reliable browser tooltip behaviour)
    • 





v16.2


New Features


    

  • Parallel job execution — all RunPod jobs are submitted simultaneously and the results table populates when complete; toggle in Configure. A "live cells" sub-toggle updates each cell as results arrive instead of waiting for the full batch
    • 

  • Auto-retry on parse failure — jobs returning "No JSON in response" are automatically retried up to 2 times before being marked as errors
    • 

  • Retry Failed button — after a run, any still-erroring cells can be resubmitted individually without re-running the full batch; button appears in the footer only when failed results are present; respects parallel/serial mode
    • 





v16.1.1


Bug Fix


    

  • Full transcript now sent to RunPod — transcripts were being silently truncated to 4,000 characters before submission, causing different results vs other applications using the same endpoint; now the complete transcript text is sent
    • 





v16.1


Improvement


    

  • Persistent prompt selections in Test Runner — selected prompts (e.g. Q5, Q8) now stay checked across runs, drawer close/reopen, and tab switches; selections only change on explicit user action; deleted cards are automatically removed from the selection
    • 





v16


New Features


    

  • Collapsed card-type badge — a letter badge (P/B/D/F/C) appears in the card header when collapsed, colored with each type's accent; hidden when expanded; updates automatically on type switch
    • 

  • Draggable rows — prompt cards can now be dragged to reorder using HTML5 drag-and-drop; drag the grip handle on the left of any card header
    • 

  • Semantic versioning in changelog — "Build X ·" prefix and date labels removed; entries now use version numbers only (Major.Minor.Patch)
    • 

  • Dependency source injection — when a Dependency card has source questions with Q IDs assigned, the output_rule automatically appends a directive instructing the model to retrieve and use those source question results directly, with multi-source matching instructions
    • 





v15


Dependency Rule — Source Results Are Ground Truth


  • Rewrote the base DEPENDENCY_RULE constant to explicitly instruct the model that source question results are already computed and must be treated as ground truth
  • Model will no longer re-evaluate the transcript to answer source questions
  • Previous wording ("if applicable") gave the model an out to ignore dependency results and reason from scratch
  • New rule: match source results against source_conditions directly; only evaluate the transcript if required_behaviors are explicitly specified alongside source conditions
  • Applies automatically to all dependency cards — no per-card changes needed




v14


Additional Instructions


  • Added Additional Instructions textarea to the bottom of all five card types (Phrase, Behavior, Dependency, Filter, Classification)
  • Content is appended to the card's output_rule in the JSON output — no new field, just enriches the existing rule string
  • Preview's Output Rule section shows the base rule + addition in accent italic so the two are visually distinct
  • Fully persisted with session save/restore


Preview — Output Rule Section


  • All five card previews now show a collapsible Output Rule section at the bottom
  • Collapsed by default — click › OUTPUT RULE to expand
  • Shows the exact rule text the evaluator will receive, including any Additional Instructions appended


Bug Fix — Text Selection in Behavior / Cue Textareas


  • Selecting text inside Behavior and Detection Cues textareas was incorrectly triggering drag-to-reorder
  • Root cause: rows had draggable="true" baked in, which the browser intercepts before any JS event can stop it
  • Fix: rows are no longer draggable by default; the drag handle's mousedown temporarily enables dragging on the parent row, and dragend clears it — text selection now works normally


Bug Fix — Changelog Modal Scrolling


  • Changelog modal content was spilling out instead of scrolling
  • Root cause: the markdown-to-HTML converter emitted a stray </div> (from a --- separator) that closed the scroll container before any content entered it
  • Fixed converter to skip separators, and fully scoped the modal with inline styles immune to CSS cascade conflicts from other modal definitions in the stylesheet
